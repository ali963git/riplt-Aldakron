# الذَّاكِرُون | Aldakron — Backend API

خادم API الخلفي لمنصة **الذَّاكِرُون | Aldakron**، منصة رقمية إسلامية تجمع القرآن الكريم، الأذكار، مواقيت الصلاة، اتجاه القبلة، ونظام مستخدمين متكامل.

> هذا هو مكوّن الـ **Backend** فقط ضمن معمارية المشروع (`backend/` من الهيكلة الكاملة). الواجهة الأمامية (Next.js) تُبنى كمكوّن منفصل.

## المكدس التقني (Tech Stack)

| الطبقة | التقنية |
|---|---|
| Runtime | Node.js 20+ |
| الإطار | NestJS 10 (TypeScript) |
| قاعدة البيانات | PostgreSQL 16 |
| ORM | Prisma 5 |
| المصادقة | JWT (access + refresh rotation) + bcrypt |
| التوثيق | Swagger / OpenAPI (`/api/docs`) |
| الحاويات | Docker + Docker Compose |
| CI/CD | GitHub Actions |

## مصادر البيانات الحقيقية المستخدمة

لا يحتوي المشروع على بيانات وهمية للمحتوى الديني الأساسي — بل يتصل بخدمات حقيقية موثوقة:

- **القرآن الكريم، التلاوات، التفسير، والبحث**: [Al-Quran Cloud API](https://alquran.cloud/api) (مجاني، مفتوح، معتمد على نصوص موثوقة)
- **مواقيت الصلاة، اتجاه القبلة، والتاريخ الهجري**: [Aladhan API](https://aladhan.com/prayer-times-api)

المحتوى الوحيد المُخزَّن محليًا هو: الأذكار (نصوص ثابتة موثّقة تُدار عبر Prisma seed)، وكل بيانات المستخدمين (حسابات، علامات مرجعية، تقدّم الأذكار، إعدادات الصلاة).

## الميزات المُنفَّذة في هذه المرحلة

- ✅ نظام مصادقة كامل: تسجيل، دخول، تجديد الجلسة (refresh token rotation)، خروج
- ✅ حماية كلمات المرور بـ bcrypt (12 rounds) وتخزين refresh tokens مُجزّأة (SHA-256) وليست نصًا صريحًا
- ✅ Rate limiting عام + حدود أشد صرامة على `/auth/login` و`/auth/register`
- ✅ ملف شخصي للمستخدم (عرض/تعديل)
- ✅ قسم القرآن: عرض السور، البحث، التفسير، التلاوات الصوتية، العلامات المرجعية، المفضلة، حفظ آخر قراءة
- ✅ قسم الأذكار: تصنيفات (صباح/مساء/نوم/بعد الصلاة)، عداد تسبيح ذكي بتقدّم يومي محفوظ لكل مستخدم
- ✅ قسم الصلاة والقبلة: مواقيت الصلاة حسب الإحداثيات وطريقة الحساب، اتجاه القبلة، التاريخ الهجري، حفظ إعدادات المستخدم
- ✅ رؤوس أمان (Helmet/CSP)، فحص مدخلات صارم (class-validator)، فلتر أخطاء موحّد لا يُسرّب تفاصيل داخلية
- ✅ Health check endpoint لأدوات المراقبة والموازنات
- ✅ لوحة تحكم Admin API: إدارة مستخدمين (بحث/ترشيح/تعديل دور وحالة)، إدارة محتوى الأذكار (تصنيفات وعناصر)، بث إشعارات، إحصائيات لوحة تحكم، وسجل تدقيق (`AdminAuditLog`) تلقائي لكل عملية تعديل/حذف

## الميزات غير المُنفَّذة بعد (خارج نطاق هذا التسليم)

هذه تحتاج تسليمات لاحقة منفصلة (كما اتفقنا على التسليم التدريجي):

- ⏳ واجهة أمامية للوحة الإدارة (هذا تسليم الـ API فقط؛ شاشات React/Next.js لم تُبنَ)
- ⏳ المساعد الإسلامي الذكي (AI Assistant)
- ⏳ Push Notifications الفعلية (نموذج `DeviceToken`/`Notification` وواجهة البث الإداري جاهزة، لكن التكامل الفعلي مع FCM/Web Push للإرسال الحقيقي للأجهزة لم يُنفَّذ بعد)
- ⏳ WebSocket للتحديثات اللحظية

## البدء السريع

راجع [`INSTALLATION.md`](./INSTALLATION.md) لخطوات التثبيت الكاملة، و[`docs/API.md`](./docs/API.md) لتوثيق نقاط النهاية.

```bash
cp .env.example .env      # عدّل القيم قبل المتابعة
docker compose up -d postgres
npm install
npx prisma migrate dev
npx prisma db seed
npm run start:dev
```

الخادم يعمل على `http://localhost:4000/api`، والتوثيق التفاعلي على `http://localhost:4000/api/docs`.

```
  schema.prisma
  seed.ts
```

## الأمان — ملخّص القرارات

- **لا كلمات مرور نصية أبدًا**: bcrypt بـ 12 جولة تجزئة.
- **Refresh tokens مُجزّأة**: تسريب قاعدة البيانات وحده لا يكفي لانتحال جلسة.
- **Rate limiting متدرّج**: حد عام 100 طلب/دقيقة، وحد أشد (5-10 طلبات/دقيقة) على مسارات المصادقة الحساسة.
- **Validation صارم**: `whitelist` + `forbidNonWhitelisted` يرفضان أي حقل غير متوقع في الطلب (دفاع ضد mass-assignment).
- **رسائل خطأ لا تُسرّب معلومات**: لا فروقات بين "بريد غير موجود" و"كلمة مرور خاطئة"، ولا تُعرض stack traces للعميل أبدًا.
- **CORS بقائمة سماح صريحة فقط** — لا `*` في الإنتاج.
